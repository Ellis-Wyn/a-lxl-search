## 产品概述

### 🎯 产品定位

**病理AI药研情报库**是一个专注于药物研发第一手资讯的智能数据库系统，帮助研发人员快速掌握：
- 进入临床 I/II/III 期的靶点与分子
- CDE 受理/审评相关公开信息
- 药企管线竞争格局
- PubMed 最新文献进展

### 📊 当前数据规模

| 数据类型 | 数量 | 来源 |
|---------|------|------|
| 研发管线 | 167条 | 12家药企官网 |
| 靶点信息 | 7个 | 数据库存储 |
| PubMed文献 | 实时搜索 | NCBI Entrez API |
| 药企覆盖 | 14家公司 | 官网爬虫 |

### 🏆 核心优势

- ✅ **实时更新**: 每日凌晨2点自动爬取最新数据
- ✅ **智能搜索**: 支持公司名称别名、中英文名称、简称搜索
- ✅ **数据归一化**: 自动标准化研发阶段、适应症、公司名称
- ✅ **来源追溯**: 每条数据都可追溯到原始来源
- ✅ **竞品监控**: 自动检测Phase Jump和竞品退场

## 核心功能

### 1️⃣ 管线查询与管理

**功能描述**: 查询和管理药企研发管线数据

**主要特性**:
- 支持按关键词搜索（药物代码、适应症、公司名称）
- 公司名称智能映射（支持35家公司的别名、简称）
- 研发阶段标准化（临床前、Phase I/II/III等）
- 管线统计分析（按公司、按阶段分布）

**API示例**:
```bash
# 搜索管线
GET /api/pipeline/search?keyword=EGFR

# 按公司查询
GET /api/pipeline/company/恒瑞医药

# 统计信息
GET /api/pipeline/statistics
```

---

### 2️⃣ PubMed文献智能搜索

**功能描述**: 智能搜索PubMed文献并自动排序

**主要特性**:
- **智能查询转换**: EGFR → 专业检索式（同义词扩展）
- **多维度排序**:
  - 时效性权重 70%
  - 临床数据披露 +50分
  - Phase III/监管认定 +40分
  - 高质量来源 +30分
- **关键指标提取**: 自动识别ORR、PFS、OS、n=等临床指标
- **作用机制识别**: 小分子/单抗/ADC/PROTAC等

**API示例**:
```bash
# 搜索文献
POST /api/pubmed/search
{
  "target_name": "EGFR",
  "max_results": 50,
  "keywords": ["inhibitor", "TKI"],
  "diseases": ["lung cancer"],
  "include_clinical_trials": true
}
```

---

### 3️⃣ 爬虫调度系统

**功能描述**: 定时自动爬取药企官网和CDE平台数据

**主要特性**:
- **定时调度**: 每天凌晨2点自动运行
- **并发控制**: 最多3个爬虫同时运行
- **智能重试**: 失败自动重试，指数退避
- **执行日志**: 记录每次运行的详细日志
- **健康监控**: 实时监控爬虫运行状态

**爬虫覆盖**:
- 12家药企官网（恒瑞、百济、信达、君实等）
- CDE平台（临床试验登记）
- 共14个爬虫

**API示例**:
```bash
# 查看调度器状态
GET /api/crawlers/status

# 手动触发所有爬虫
POST /api/crawlers/trigger

# 查看执行历史
GET /api/crawlers/executions?days=7
```

---

### 4️⃣ 数据归一化服务

**功能描述**: 自动标准化各类数据

**归一化类型**:
- **公司名称**: "恒瑞" → "恒瑞医药"（35家公司映射）
- **研发阶段**: "临床I期" → "Phase 1"
- **适应症**: "非小细胞肺癌" → "NSCLC"
- **靶点别名**: HER2 = ERBB2 = c-ErbB2

---

### 5️⃣ 竞品监控与预警

**功能描述**: 自动检测竞争格局变化

**监控类型**:
- **Phase Jump检测**: Phase II → Phase III 自动告警
- **新入局者识别**: 公司首次进入某靶点
- **竞品退场预警**: 连续21天未抓取到标记终止

## 功能模块详解

### 1. PubMed智能搜索模块

**核心算法**: 多维度评分系统

```python
# 评分算法（scoring_algorithms.py）
总分 = 时效性得分(70%)
     + 临床数据得分(15%)
     + 阶段得分(5%)
     + 监管认定得分(5%)
     + 来源质量得分(5%)
     - 惩罚得分

# 时效性得分（0-100分）
0-30天:    100分
31-90天:   80分
91-365天:  60分
366-730天: 40分
>730天:    20分

# 临床数据加分
有ORR/PFS/OS: +50分
有样本量(n=):  +25分
有p-value:    +25分

# 监管认定加分
Breakthrough Therapy: +30分
Fast Track:          +20分
Orphan Drug:         +20分
First-in-class:      +30分

# 来源质量加分
ASCO/AACR/NEJM/Lancet: +30分
JCO/Lancet Oncology:  +20分
Nature/Science/Cell:  +20分
```

**智能查询转换**:
```python
# 用户输入
"EGFR"

# 自动转换为
("EGFR"[Gene/Protein Name] OR "ERBB1"[Gene/Protein Name] OR "HER1"[Gene/Protein Name])
AND (Clinical Trial[Filter] OR "Phase")
AND ("inhibitor"[Ti/Ab] OR "tyrosine kinase"[Ti/Ab])
AND ("Lung Neoplasms"[MeSH Terms])
AND ("2025/02/07"[Date - Publication] : "2026/02/07"[Date - Publication])
```

## FAQ常见问题

### Q1: 如何修改定时爬取时间？

**A**: 编辑 `.env` 文件：
```bash
CRAWLER_SCHEDULER_TIME=06:00  # 改为早上6点
```

然后重启服务器。

---

### Q2: 数据库密码忘记了怎么办？

**A**: PostgreSQL重置密码：
```bash
# Windows
# 1. 打开pgAdmin，重置密码
# 2. 修改 .env 文件中的 DB_PASSWORD
```

---

### Q3: 如何添加新的药企爬虫？

**A**: 步骤：
1. 复制 `crawlers/hengrui_spider.py` 作为模板
2. 修改公司名称和URL
3. 在 `crawlers/__init__.py` 中注册
4. 测试爬虫

参考文档：`docs/CRAWLER_DEVELOPMENT_GUIDE.md`

---

### Q4: PubMed搜索没有结果怎么办？

**A**: 检查：
1. 网络连接（NCBI API需要外网）
2. API密钥（可选，但推荐配置）
3. 查看日志：`logs/pathology_ai.log`

---

### Q5: 如何备份数据？

**A**: PostgreSQL备份：
```bash
# 备份
pg_dump -U postgres drug_intelligence_db > backup_$(date +%Y%m%d).sql

# 恢复
psql -U postgres drug_intelligence_db < backup_20260207.sql
```

- **内存**: 约200-500MB（Python进程）

- **磁盘**: 数据库约50MB，日志文件约5MB
- **网络**: 爬虫运行时约1-5Mbps
